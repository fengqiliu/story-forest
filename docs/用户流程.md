## 一、核心用户旅程

### 1.1 新用户首次使用流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        新用户首次使用                            │
└──────────────────────────────────────────────────────────────────┘

  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
  │ 下载打开 │ -> │ 授权登录 │ -> │ 选择年龄 │ -> │ 进入首页 │
  │  App     │    │ 微信登录 │    │ 3-10 岁   │    │ 开始探索 │
  └─────────┘    └─────────┘    └─────────┘    └─────────┘
       │              │              │              │
       ▼              ▼              ▼              ▼
  展示品牌        获取用户        匹配内容        推荐今日
  形象和         信息建立        年龄标签        精选故事
  产品定位        账户

详细步骤:
1. 启动页 (2 秒) → 品牌展示
2. 授权页 → 点击「微信一键登录」→ 获取 openid
3. 年龄选择 → 选择年龄段 → 保存用户画像
4. 首页 → 展示今日精选 → 开始浏览
```

### 4.2 答题数据流

```
┌──────────────────────────────────────────────────────────────────┐
│                        答题数据流                                │
└──────────────────────────────────────────────────────────────────┘

1. 获取题目
   前端 ──GET /api/quiz/questions?story_id=123──► 后端
   前端 ◄──{questions: [...], total: 5}───────── 后端

2. 提交答案
   前端 ──POST /api/quiz/submit ──► 后端
   {
     question_id: 456,
     user_answer: ["B"],
     time_spent: 15
   }

3. 返回结果
   前端 ◄── {is_correct: true, points: 15, explanation: "..."} ── 后端

4. 更新积分
   前端 ── 自动更新本地积分显示
   后端 ── 异步写入积分流水
```

### 4.3 配音上传流程

```
┌──────────────────────────────────────────────────────────────────┐
│                        配音上传流程                              │
└──────────────────────────────────────────────────────────────────┘

1. 录音
   用户 ──点击录制──► 前端调用录音 API ──► 生成临时音频文件

2. 预处理
   前端 ──音频剪辑──► 截取有效部分 (去除前后空白)
   前端 ──音量标准化──► 调整到目标响度 (-16 LUFS)

3. 上传
   前端 ──POST 音频到 COS──► 获取音频 URL
          (直传腾讯云)

4. 提交记录
   前端 ──POST /api/dubbing/upload ──► 后端
   {
     story_id: 123,
     character_name: "灰姑娘",
     audio_url: "https://cos.xxx.com/xxx.mp3",
     duration: 30
   }

5. 审核 (异步)
   后端 ──加入审核队列──► AI 审核 (敏感内容检测)
   审核通过 ──更新 status=1──► 前端可见
   审核拒绝 ──更新 status=2──► 通知用户
```

## 八、关键指标定义

### 8.1 漏斗分析

```
用户激活漏斗:

下载打开                    100%
    │
    ▼
完成微信登录                 85%  (-15% 流失)
    │
    ▼
选择年龄段                   95%  (-5% 流失)
    │
    ▼
完成首次阅读                 70%  (-25% 流失)
    │
    ▼
完成首次答题                 40%  (-30% 流失)
    │
    ▼
次日留存                     50%

优化重点:
1. 首次阅读完成率 - 优化新手引导
2. 答题转化率 - 降低首次答题难度
```

---

### 8.2 核心指标定义

| 指标 | 定义 | 计算方式 | 目标值 |
|------|------|----------|--------|
| DAU | 日活跃用户 | 当日打开过 App 的去重用户数 | 10,000 |
| 次日留存 | Day1 留存 | 新增用户次日活跃占比 | 50% |
| 7 日留存 | Day7 留存 | 新增用户 7 日后活跃占比 | 30% |
| 人均时长 | 平均使用时长 | 日总时长 / DAU | 20 分钟 |
| 完读率 | 故事完成率 | 完成故事数 / 开始故事数 | 60% |
| 答题参与率 | 答题渗透率 | 参与答题用户 / 总用户 | 40% |
| 付费转化率 | VIP 转化 | 付费用户 / 总用户 | 5% |
